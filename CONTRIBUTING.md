# Οδηγός Συνεργασίας & Χρήσης GitHub 🚀

Καλώς ήρθατε στο repository της ομάδας μας! Χρησιμοποιούμε το GitHub για να έχουμε οργανωμένο τον κώδικά μας και να μπορούμε να γράψουμε την τελική έκθεση εύκολα.

## 📌 Γιατί χρησιμοποιούμε το GitHub;
1. **Ασφάλεια:** Κάθε αλλαγή αποθηκεύεται. Αν κάτι "σπάσει", μπορούμε να γυρίσουμε πίσω.
2. **Διαφάνεια:** Φαίνεται ποιος έκανε τι και πότε (απαραίτητο για το Changelog της έκθεσης).
3. **Συνεργασία:** Δουλεύουμε όλοι στο ίδιο project χωρίς να μπερδεύουμε τα αρχεία μας.

## 💻 Βασικές Εντολές (Terminal)
Πριν ξεκινήσετε την εργασία σας, ανοίξτε το terminal στο φάκελο του project και ακολουθήστε αυτά τα βήματα:

1. **`git clone https://github.com/Kotsalas/hou-plhpro-team-project-cooking-recipes.git`** <b>ΜΟΝΟ</b> την πρώτη φορά ώστε να μπορέσουμε να *αντιγράψουμε* το πρότζεκτ στον υπολογιστή μας. (η επανάληψη αυτής της εντολής είναι απαραίτητη μόνο εάν καταλάθος διαγράψουμε το αρχείο του πρότζεκτ από την συσκευή μας)

2. **`git pull`** *Το κάνουμε ΠΑΝΤΑ στην αρχή. (εκτός απο την πρώτη φορά)* Κατεβάζει τις τελευταίες αλλαγές που έκαναν οι υπόλοιποι.

3. **`git add --all`** Προετοιμάζει τα αρχεία που αλλάξατε για να "ανέβουν".

4. **`git commit -m "Περιγραφή αλλαγής"`** Οριστικοποιεί τις αλλαγές στον υπολογιστή σας. Γράψτε σύντομα τι κάνατε (π.χ. "Πρόσθεσα 3 συνταγές").

5. **`git push`** Στέλνει τις αλλαγές σας στο GitHub για να τις δούμε όλοι.

## ⚠️ Κανόνες για να μην έχουμε προβλήματα
* **Ενημέρωση:** Πριν ξεκινήσετε να αλλάζετε ένα αρχείο, γράψτε στο chat της ομάδας "Ξεκινάω να δουλεύω στο αρχείο X".
* **Συχνά Commits:** Μην περιμένετε 3 μέρες για να κάνετε push. Κάντε μικρές και συχνές αλλαγές.
* **Changelog:** Μετά από κάθε σημαντική δουλειά, ενημερώστε το αρχείο `CHANGELOG.md`.
